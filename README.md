# PRIMUM — PWA оценки автосервисов

Опрос водителей → запись в Google Sheets. Работает офлайн, ставится на домашний экран Android/iOS.

## Что в комплекте
- `index.html`, `app.js`, `sw.js`, `manifest.webmanifest`, `icon-*.png` — само PWA
- `Code.gs` — серверная часть (Google Apps Script)
- `PRIMUM_template.xlsx` — шаблон таблицы (листы `Fleet` и `Responses`)

## Как связаны части
```
Телефон (PWA)  ──GET token──▶  Apps Script  ──▶  лист Fleet   (база ТС для автокомплита)
               ──POST ответ─▶               ──▶  лист Responses (ответы водителей)
```

## Развёртывание (10 минут)

### 1. Таблица
1. Загрузите `PRIMUM_template.xlsx` на Google Диск, откройте через Google Таблицы (или File → Import).
2. Лист `Fleet` замените реальным автопарком: колонки `number` (номер), `type` (`tractor`/`trailer`), `active` (`yes`/`no`). Данные из Excel/CSV можно вставить прямо сюда.
3. Лист `Responses` не трогайте — туда пишутся ответы.

### 2. Apps Script
1. В таблице: **Расширения → Apps Script**.
2. Удалите пустой код, вставьте содержимое `Code.gs`, сохраните.
3. В строке `SHARED_TOKEN` замените `CHANGE_ME_primum_2026` на свой набор символов.
4. **Развернуть → Новое развёртывание → Веб-приложение**:
   - Выполнять от имени: **Я**
   - Доступ: **Все**
5. Скопируйте выданный **URL веб-приложения**.

### 3. PWA
1. В `app.js` (вверху, объект `CONFIG`):
   - `API_URL` — вставьте URL веб-приложения из шага 2.5
   - `SHARED_TOKEN` — тот же токен, что в `Code.gs`
2. Загрузите все файлы PWA на любой HTTPS-хостинг (обязательно HTTPS, иначе service worker и «установка» не работают). Подойдёт GitHub Pages, Netlify, ваш внутренний сервер.
3. Откройте адрес на телефоне → браузер предложит «Добавить на главный экран».

## Проверка
- Открыть на телефоне, начать вводить номер тягача → появляется список из `Fleet`.
- Заполнить опрос, отправить → строка появляется в `Responses`.
- Включить авиарежим, отправить ответ → «ответ сохранён», после возврата сети досылается сам.

## Что заложено
- **Автокомплит** тягача и прицепа из базы `Fleet` (независимые поля; прицеп необязателен).
- **Офлайн**: оболочка кэшируется, ответы копятся в IndexedDB, Background Sync досылает их при появлении сети. Дедупликация по `client_id` — повторной строки не будет.
- **Валидация** дублируется на сервере (сервис из списка, оценка 0–10, комментарий обязателен).

## Если ответы не доходят

1. Откройте приложение в браузере на компьютере, нажмите **F12 → Console**.
2. При загрузке должно появиться `[PRIMUM] База ТС загружена: тягачей N, прицепов M`.
   - `Неверный SHARED_TOKEN` — токен в `app.js` не совпадает с `Code.gs`.
   - `Сервер недоступен` — проверьте развёртывание Apps Script (доступ «Все»).
3. Проверка скрипта в обход приложения: откройте в браузере
   `URL_ВЕБ_ПРИЛОЖЕНИЯ?token=ВАШ_ТОКЕН` — должно вернуться `{"ok":true,"tractors":[...]}`.

**После правки `Code.gs`** обязательно: Развернуть → Управление развёртываниями →
карандаш → Версия: **Новая версия** → Развернуть. Простого сохранения недостаточно.

**После правки файлов приложения** поднимайте версию кэша в `sw.js`
(`primum-shell-v2` → `v3`), иначе телефоны останутся на старой версии.

## Ограничения (важно понимать)
- `SHARED_TOKEN` в клиенте виден любому, кто откроет код страницы. Это барьер от случайного мусора, **не** криптозащита. Для внутреннего инструмента приемлемо; если нужна настоящая аутентификация — потребуется бэкенд с сессиями.
- Apps Script Web App имеет квоты Google (лимиты на число запросов в сутки). Для опроса водителей их с запасом хватает, но это не промышленная нагрузка.
- iOS ограничивает Background Sync: досылка там надёжнее срабатывает при следующем открытии приложения, а не строго в фоне.
